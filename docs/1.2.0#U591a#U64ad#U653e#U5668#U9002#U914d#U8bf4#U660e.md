# 1.2.0 多播放器适配说明

本版在 1.1.3 多 AI 连接版基础上，加入 QQ 音乐、酷狗音乐、网易云音乐的多播放器基础适配。

## 支持范围

| 播放器 | Android 包名 | 支持内容 |
| --- | --- | --- |
| QQ 音乐 | `com.tencent.qqmusic` | 媒体状态、播放/暂停、上一首/下一首、进度跳转、界面歌词读取、OCR 兜底 |
| 酷狗音乐 | `com.kugou.android` | 媒体状态、播放/暂停、上一首/下一首、进度跳转；歌词走 LRCLIB 匹配 |
| 网易云音乐 | `com.netease.cloudmusic` | 媒体状态、播放/暂停、上一首/下一首、进度跳转；歌词走 LRCLIB 匹配 |

## 本次改动

- Android 通知监听服务增加 `PlayerCatalog`，集中管理播放器包名和显示名称。
- 当前媒体会话选择逻辑改为：优先选择正在播放的受支持播放器，其次选择受支持播放器，再回退到其他音乐类媒体会话。
- App 首页、诊断信息、网页遥控器和 MCP / REST 返回都能看到当前播放器名称。
- 空状态提示从“请先在 QQ 音乐播放一首歌”改为“请先在 QQ 音乐、酷狗音乐或网易云音乐播放一首歌”。
- 服务端 `PlaybackSnapshot` 增加可选 `playerName` 字段，并可从 `packageName` 自动推导。
- 版本号升级为 1.2.0，Actions 产物名升级为 `tongpin-clean-1.3.0-public-apk`。

## 还没做的内容

- 酷狗 / 网易云自动搜索点歌尚未开放。原因是自动点歌依赖无障碍读取和点击具体界面，不同 App 的搜索入口、结果列表、广告弹窗都不一样，需要单独适配和真机测试。
- 酷狗 / 网易云界面歌词读取尚未开放。本版先使用系统媒体会话与 LRCLIB 时间轴歌词。
- 进度跳转依赖播放器是否开放 MediaSession seek 能力；如果某个播放器不响应，可先使用播放/暂停/切歌。

## 测试建议

1. 分别在 QQ 音乐、酷狗音乐、网易云音乐播放歌曲。
2. 回到同频 Clean，点击“刷新状态”。
3. 检查首页是否显示正确歌名、歌手、当前播放器。
4. 用 App 本地按钮、网页遥控器或 MCP 工具测试播放/暂停/上一首/下一首。
5. 测试进度跳转，如果失败，记录手机型号、Android 版本、播放器版本和房间诊断信息。
