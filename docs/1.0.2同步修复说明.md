# 同频 Clean 1.0.2-sync 修复说明

## 同步延迟

- 命令轮询约每 0.85 秒执行一次。
- MediaSession 的歌曲或播放状态变化会立即触发状态上报。
- 播放控制后在 0.12、0.42、0.9、1.6、2.7、4.2 秒附近追加状态抓取，覆盖 QQ 音乐延迟切歌的情况。
- 普通进度上报做了去重与节流，避免每次轮询都写一次服务器数据文件。
- 服务端返回播放中状态时，会根据最近一次服务器接收时间做最多 5 秒的进度估算。

## 刷新按钮

“立即刷新状态”现在会：

1. 重新查找当前活跃的 QQ 音乐 MediaSession；
2. 立即读取歌曲、进度和播放状态；
3. 强制上传一次房间状态；
4. 等待真实上传完成后显示“刷新成功”，超过约 5 秒显示“刷新超时”。

## 同步歌词

- Android 端使用 LRCLIB 的 `/api/get-cached`、`/api/search` 与 `/api/get` 接口匹配歌曲。
- 解析带时间戳的 LRC，并按当前播放位置计算当前句与下一句。
- 房间状态新增 `lyric`、`nextLyric`、`lyricsSource`、`lyricsSynced` 字段。
- MCP `get_room` 与网页遥控器都能读取这些字段。
- 只同步当前句和下一句，不上传整份歌词。

## 命令可靠性

- 新增命令执行中的去重状态。
- 手机执行完命令后保存最终状态；若最终 ACK 因网络失败没有送达，会重试同一最终状态。
- 不会再把尚未执行或执行失败的命令直接补报为 executed。

## 验证

- 服务端 TypeScript 编译通过。
- 服务端 3 项自动测试通过：命令生命周期、歌词字段持久化、播放进度估算。
- Android Java 源文件语法解析通过。
- APK 仍需由 GitHub Actions 使用 Android SDK 完成最终构建验证。
